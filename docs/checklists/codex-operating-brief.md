# Codex Operating Brief (Non-Negotiable)

Use this brief at the start of every task:
1. Inspect repository state before making changes.
2. Do not assume file presence or architecture details.
3. Keep `main` buildable and stable.
4. Prefer simple, production-safe implementations.
5. Add tests for non-trivial changes.
6. Update docs whenever behavior or architecture changes.
7. Ship one clean PR-sized unit per day.
8. Remove dead code and stale TODOs.
9. Do not broaden scope beyond the assigned day.
